# 대화 액션 대원칙 (A/B)

본 원칙은 파이프라인 단계와 무관하게, 모든 대화 응답 설계에 우선 적용한다.

## A 케이스: 결론이 유일한 경우
- 결론만 말하고 끝내지 않는다.
- 반드시 다음 액션 선택지를 함께 제공한다.
- 선택지는 최소 2개로 구성한다.
  - 서비스 다음 액션 (예: `재입고 알림 신청`)
  - `대화 종료`

## B 케이스: 결론에 선택이 필요한 경우
- 사용자가 바로 선택할 수 있게 선택지를 제공한다.
- 사용자가 선택한 뒤 결과가 유일해지면 A 케이스로 전환한다.
- 결과가 여전히 복수면 B 케이스를 반복한다.

## 버튼 우선
- 문장 입력이 필수가 아닌 단계는 버튼/퀵리플라이를 우선 제공한다.
- 숫자(예: D-45)처럼 단독으로 의미가 없는 값은 선택지로 노출하지 않는다.
- 선택지는 사용자가 즉시 실행 가능한 값만 제공한다.

## 모호 요청 처리 (의도 혼합)
- 한 문장에 서비스 의도가 2개 이상 섞이면 모호 요청으로 판단한다.
- 모호 요청은 즉시 수행하지 않고 의도 분해 선택지를 먼저 제시한다.
  - 예: `재입고 일정 안내`, `일반 문의`
- 선택지는 복수 선택을 허용한다. (예: `1,2`)
- 복수 선택된 의도는 사용자가 선택한 순서대로 처리한다.
