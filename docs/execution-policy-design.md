# Widget Load Contract Redesign & Execution Policy (2026-03-10)

## 목적
본 문서는 위젯 로드 방식 재설계의 계약(Contract) 정의와, 구현 단계에서 반드시 지켜야 할 실행 정책을 명시한다.
이 문서는 재설계의 기준 문서이며, 상세 구현 설계는 `docs/widget-load-redesign.md`에 정리한다.

## 재설계 계약 (Contract)
1. `B_chat_widgets`는 `page_keys` 개념을 폐지하고, 인증은 `B_chat_widgets.id` + `B_chat_widgets.public_key` 일치로만 판단한다.
2. `B_chat_widgets`는 위 조건이 맞으면 어디서든 호출 가능하며, 다중 호출도 허용한다.
3. `B_chat_widget_instances`는 `id` + `public_key` + `template_id`로 호출 가능해야 한다.
4. `B_chat_widget_instances`의 인증 권한 확인은 `id` + `public_key`로만 수행한다.
5. `B_chat_widget_instances`의 위젯 출력 방식은 `template_id`로 결정한다.
6. `B_chat_widget_instances`는 `(id, public_key, template_id)` 조합이 1:1로 보장되며 사실상 `id`에 템플릿이 귀속된다.
7. `B_chat_widget_instances`는 위 조건이 맞으면 어디서든 호출 가능하며, 다중 호출도 허용한다.

## 계약 해석 및 강제 규칙
1. 템플릿(=`B_chat_widgets`) 호출은 `id + public_key`가 유일한 인증 수단이며, 도메인/경로 제약을 적용하지 않는다.
2. 인스턴스(=`B_chat_widget_instances`) 호출은 `id + public_key + template_id`를 입력으로 받되, 인증은 `id + public_key`로만 수행한다.
3. 인스턴스 호출 시 제공된 `template_id`는 반드시 인스턴스가 가진 `template_id`와 일치해야 하며, 불일치 시 오류로 처리한다.
4. 인스턴스 호출의 출력(렌더링/정책/테마)은 `template_id`에 의해 결정된다.
5. 다중 호출 허용은 동일 키 조합에 대한 반복 초기화 요청이 허용됨을 의미하며, 서버는 요청을 거절하지 않는다.
