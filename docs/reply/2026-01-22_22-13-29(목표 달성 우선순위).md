# 답변
요청하신 검증(“실제 통화 → LLM → KB → TTS”)을 **최우선**으로 올렸습니다. 체크리스트 Phase 3의 맨 위에 최우선 검증 항목을 추가해 우선순위를 재정렬했습니다.

다음에 할 일(최우선)
1) 실제 통화 수신을 위한 전화 Provider(Twilio/Telnyx) 연동 결정 및 Webhook 확보
2) 통화 세션 생성 시 어떤 에이전트/KB를 붙일지 라우팅 규칙 확정
3) LLM 호출 시 KB 검색(RAG) 결과를 포함한 프롬프트 구성 구현
4) LLM 응답을 TTS로 변환해 통화로 송출하는 파이프라인 연결
5) 위 1~4가 완료되면 실제 전화로 테스트 진행

현재 코드 베이스에는 WebSocket 기반 웹 입력(/call/[token]) UI만 있고, 실제 전화 Provider 연동 및 TTS 송출은 아직 없습니다. 그래서 **다음 단계는 전화 Provider 선택과 Webhook 연동**입니다.

필요 정보
- 사용할 전화 Provider(Twilio/Telnyx 등)
- 통화 수신 Webhook URL
- LLM/TTS 엔진 및 API 키

이 정보를 주시면 “실제 전화 → LLM → KB → TTS” 경로를 바로 연결하겠습니다.
