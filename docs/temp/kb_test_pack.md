# KB Test Pack (MCP 전체 기능 검증용)

아래 문서를 그대로 KB에 등록하면, Playground에서 MCP 기능(주문 조회/배송 조회/문의 티켓 생성)을 모두 테스트할 수 있습니다.

---

## 문서 1: 주문 정보 변경 안내 (필수)
제목: 주문 정보 변경 안내
카테고리: 주문/변경
내용:
1) 주문 정보 변경 가능 항목
- 배송지 주소 변경
- 수령인 이름/연락처 변경
- 배송 요청사항 변경

2) 변경 가능 조건
- 결제 완료 후에도 “배송 준비 중” 상태까지 변경 가능
- “배송 시작” 이후에는 변경 불가

3) 필요한 정보
- 주문번호
- 변경할 항목(주소/연락처/요청사항)

4) 처리 절차
- 주문번호 확인
- 변경 요청 접수(문의 티켓 생성)
- 담당자 확인 후 처리 결과 안내

---

## 문서 2: 배송 조회 안내 (필수)
제목: 배송 조회 안내
카테고리: 배송
내용:
1) 배송 상태 조회를 위해 주문번호가 필요합니다.
2) 주문번호를 받으면 배송 조회를 진행합니다.
3) 배송 상태는 “준비중/배송중/배송완료”로 안내합니다.

---

## 문서 3: 환불/취소 안내 (테스트)
제목: 환불 및 취소 안내
카테고리: 환불/취소
내용:
1) 환불/취소 요청은 주문번호 확인 후 접수됩니다.
2) 접수 후 담당자가 확인하여 처리합니다.
3) 처리 상태는 문의 티켓으로 기록됩니다.

---

## 테스트 입력 예시
1) “주문 정보를 수정하고 싶어요. 주문번호는 20260127-0000014입니다.”
2) “배송 조회 부탁해요. 주문번호 20260127-0000014”
3) “환불 요청하고 싶습니다. 주문번호 20260127-0000014”

---

## 기대 동작
1) 주문번호 인식 → MCP `lookup_order` 호출
2) 배송 관련이면 MCP `track_shipment` 호출
3) 변경/환불/취소 문의면 MCP `create_ticket` 호출
4) KB 근거 기반으로 “요약 → 근거 → 상세 → 다음 액션” 형태로 답변
