# conversation_install_code_preview_layout_design.md

## 목적
- `/app/conversation` 페이지에서 설치 코드가 사용자에게 **항상 노출**되고, **클릭 한 번으로 복사** 가능하도록 한다.
- 페이지 하단의 미리보기(preview) 항목을 **3열 그리드**로 배치한다.

## 문제 요약
- 설치 코드가 보이지 않아 사용자가 복사할 수 없음.
- 미리보기 항목이 1열 3행으로 출력됨(요구: 3열).
- 미리보기 항목별 on/off 상태와 가시성(public/user/admin) 정보를 확인할 수 없음.
- 런처 위치 프리뷰가 실제 런처 위치와 일치하지 않음.
- 런처 출력 화면의 너비를 마우스로 변경할 수 없음.

## 범위
- 대상 페이지: `http://localhost:3000/app/conversation`
- 범위 내 UI 요소:
  - 설치 코드 표시 영역
  - 설치 코드 복사 인터랙션
  - 하단 미리보기 목록 레이아웃(3열 그리드)

## 비범위(Non-goals)
- 설치 코드 생성 로직 변경
- 설치 코드 텍스트 포맷/내용 변경
- 미리보기 항목의 데이터 로딩 로직 변경

## 현재 이해(확정 전)
아래 항목은 구현 전에 **사용자 확정**이 필요하다.
- `/app/conversation` 페이지에서 설치 코드가 항상 보이고 복사 가능해야 한다.
- 페이지 하단의 미리보기 항목은 3열 그리드로 보여야 한다.
- 설치 코드의 내용 자체는 변경하지 않는다(표시/복사 UX만 개선).
- 미리보기 데이터/로직은 변경하지 않고 레이아웃만 변경한다.

## 설계 개요
### 1) 설치 코드 표시/복사 UX
- 설치 코드가 보이지 않는 원인을 파악한다:
  - 숨김 처리(`display: none`, conditional render, 권한/feature gate)
  - 레이아웃 밀림/overflow
  - 텍스트 컬러와 배경 대비 문제
- 해결 방식:
  - 설치 코드 영역을 **항상 렌더링**하도록 보장.
  - 코드 표시를 `code`/`pre` 또는 디자인 시스템의 코드 블록 컴포넌트로 고정.
  - **복사 버튼** 또는 **클릭-복사** 액션을 추가.
  - 복사 성공 시 토스트/상태 텍스트로 피드백 제공.
- 접근성:
  - `aria-label` 제공.
  - 키보드 포커스 및 `Enter/Space` 복사 지원.

### 2) 미리보기 3열 그리드
- 하단 미리보기 리스트 컨테이너를 **CSS grid** 3열로 변경.
- 반응형 고려:
  - 데스크톱: 3열 고정
  - 태블릿: 2열
  - 모바일: 1열
- 아이템 간 간격(gap)과 정렬 기준을 명시.
- 탭별 상태 표기:
  - 각 미리보기 카드에 **on/off 상태** 및 **visibility(public/user/admin)** 표시.
  - 비활성화된 탭은 **“비활성화됨”** 상태로 렌더링.

### 3) 런처 위치 프리뷰 정렬 및 리사이즈
- 런처 컨테이너 설정(`launcher.container.*`)을 기준으로 위치를 맞춘다.
  - `left`가 있고 `right`가 없으면 `bottom-left`, 그 외에는 `bottom-right`로 간주.
  - `bottom/left/right/zIndex`는 정책 값을 그대로 사용한다.
- 프리뷰 영역 너비는 **마우스 드래그로 조절 가능**해야 한다.
  - CSS `resize: horizontal` + `overflow: auto` 조합을 사용.
  - `min-width`를 두어 과도한 축소를 방지한다.

## 구현 계획 (화이트리스트 확정 후)
1. **이해확정 단계**
   - 이해 목록 제시 → 사용자 명시적 확정 수신.
2. **화이트리스트 확인**
   - 수정 가능 파일 목록을 사용자에게 받음.
   - 목록 밖 파일 수정 필요 시 즉시 중단 후 승인 요청.
3. **수정 전 백업 기록**
   - 변경 대상 파일의 수정 전 내용을 `docs/diff`에 기록.
4. **UI 수정**
   - 설치 코드 표시 영역의 렌더 조건/스타일 점검 및 수정.
   - 복사 버튼/액션 적용.
   - 미리보기 리스트 컨테이너를 3열 그리드로 변경.
5. **MCP 테스트**
   - `supabase` MCP: 백엔드/설정 영향 여부 확인(필수 호출).
   - `chrome-devtools` MCP: UI 렌더링/복사/레이아웃 확인.
6. **결과 보고**
   - 변경 요약, 테스트 결과, 롤백 경로 문서화.

## 위험 요소 및 대응
- 설치 코드 렌더 조건이 정책/권한에 의해 숨겨질 가능성 → 조건 분리 또는 정책 정의 수정 필요.
- 복사 기능이 브라우저 권한/HTTPS 제약에 영향 → fallback: 텍스트 선택 가능 상태 유지.
- 3열 레이아웃 변경 시 카드 최소 너비 부족 → 반응형 분기와 minmax 사용.
