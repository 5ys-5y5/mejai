# 답변
알겠습니다. 앞으로 reply 기록은 터미널 답변과 동일하게 남기겠습니다.

아래는 “Supabase Auth 설정(도메인/템플릿/Redirect URL) 적용”을 위한 단계별 상세 안내입니다.

1) Auth → URL Configuration
- Supabase Console → Authentication → URL Configuration
- Site URL:
  - https://mejai.help
- Redirect URLs 추가:
  - https://mejai.help/login
  - http://localhost:3000/login
- 저장

2) Auth → SMTP Settings
- Supabase Console → Authentication → SMTP Settings
- 커스텀 SMTP 정보 입력 (SendGrid/SES 등)
  - Host, Port, Username, Password
  - From Address: no-reply@mejai.help
- 설정 저장 후 “Test”로 발송 확인

3) DNS 레코드 등록
- 도메인 관리(Cloudflare/Route53 등)에서 아래 등록
  - SPF
  - DKIM
  - DMARC
- 발송 도메인 검증이 완료돼야 스팸 확률이 줄어듭니다.

4) Auth → Email Templates
- Supabase Console → Authentication → Email Templates
- “Confirm signup” 템플릿을 커스텀
  - 브랜드명 Mejai
  - 스팸함 확인 안내 문구 포함
- 저장

5) 최종 검증
- 실제 회원가입 실행
- mejai.help 도메인에서 메일 도착 확인
- 템플릿 커스텀 내용 확인
- 인증 링크 클릭 → /login 리디렉션 확인

이 단계가 완료되면 체크리스트의 아래 항목을 [x] 처리할 수 있습니다.
- Supabase Auth 설정(도메인/템플릿/Redirect URL) 적용
  - Site URL/Redirect URL 설정 완료 확인
  - 커스텀 SMTP 연결 확인
  - 커스텀 템플릿 적용 확인
