요청: mejai.help 보안(HTTPS) 경고 해결 방법 정리

상황
- 도메인 연결은 정상
- 브라우저에서 “보안 연결(HTTPS) 아님” 경고 표시

원인
- SSL 인증서가 아직 발급/적용되지 않았거나,
- 인증서가 적용될 도메인 범위(루트/서브도메인) 중 일부가 누락됨

해결 순서 (초등학생 버전)
1) Railway 대시보드에서 도메인 상태 확인
   - Custom Domains에 아래 2개가 모두 있어야 함
     - mejai.help
     - www.mejai.help
   - 상태가 “Verified / Active”인지 확인

2) DNS 설정 재확인 (Porkbun)
   - ALIAS @  → e62658wb.up.railway.app
   - CNAME www → e62658wb.up.railway.app
   - 이전 값(pixie.porkbun.com 등)은 삭제되어 있어야 함

3) 인증서 자동 발급 대기
   - 보통 10~30분 내 적용
   - 최장 72시간까지 걸릴 수 있음

4) 확인 방법
   - 브라우저에서 https://mejai.help 와 https://www.mejai.help 접속
   - 자물쇠(🔒) 표시가 보이면 성공

빠른 체크리스트
- Railway에서 두 도메인이 모두 등록되어 있다
- DNS 레코드가 Railway 값으로 되어 있다
- 10~30분 이상 대기했다

그래도 해결되지 않을 때
- Railway에서 인증서 재발급/재검증을 눌러 시도
- DNS 변경 후 캐시/전파 지연 가능성 있으므로 재대기
