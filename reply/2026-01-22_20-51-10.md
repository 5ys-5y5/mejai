# 질문
현재 단계에서 적용한 사항을 테스트하는 방법에 대해 reply 폴더에 파일을 생성하여 알려주세요

# 답변
아래 단계로 현재 적용된 기능을 테스트할 수 있습니다.

1) 로그인
- 브라우저에서 로그인 완료

2) 시뮬레이션 데이터 생성
- 브라우저 개발자 콘솔에서 실행:
```ts
const { data } = await supabase.auth.getSession();
await fetch("/api/simulate-call", {
  method: "POST",
  headers: { Authorization: `Bearer ${data.session.access_token}` },
  body: JSON.stringify({ escalated: true }),
});
```

3) 화면 확인
- /app 대시보드: 통화 수/이관률/가장 많이 호출된 에이전트 갱신
- /app/calls: 세션 목록에 신규 세션 표시
- /app/calls/{id}: 세션 상세에서 턴/ASR/요약/오디오 구간 표시
- /app/review: 이관된 경우 후속 지원 요청 생성 확인
- /app/kb: 기존 KB 목록 조회 확인 (읽기 연동)

4) API 직접 확인(선택)
- /api/sessions, /api/review-queue, /api/sessions/{id}/turns 등 호출 시 Authorization 헤더 필요
