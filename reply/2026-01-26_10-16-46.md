# 답변
도움 패널의 로그인 단계는 제거했고, 온보딩 단계는 사용자 프로필에서 org_id가 없을 때만 표시되도록 조건부 처리했습니다. `src/components/HelpPanel.tsx`

"후속 지원 요청 대상"은 더미 배열을 제거하고 `/api/review-queue`에서 실제 데이터를 불러오도록 변경했으며, 실패/로딩 상태도 표시합니다. `src/components/HelpPanel.tsx`

후속 지원 요청이 하나라도 있으면 도움 패널이 접힌 상태의 버튼 우측 상단에 빨간 점이 부드럽게 반짝이도록 추가했습니다. `src/components/HelpPanel.tsx`

/app 페이지는 Supabase 인증 상태 변경 시 데이터를 다시 불러오도록 구독을 추가해, 로그인 직후 데이터가 비어 보이던 문제를 완화했습니다. `src/app/app/page.tsx`

/app/settings의 섹션 상하 패딩을 192px에서 10px로 줄이기 위해 `p-4`를 `px-4 py-2.5`로 변경했습니다. `src/app/app/settings/page.tsx`

도움 패널은 모든 유저에게 항상 표시되도록 enabled 토글을 무시하고 항상 true로 고정했습니다(2번 온보딩 항목만 org_id 유무로 조건부 표시). `src/hooks/useHelpPanel.ts` `src/components/HelpPanel.tsx`

프로필/워크스페이스 탭에서 남아있던 섹션 패딩도 동일하게 10px로 맞췄습니다. `src/app/app/settings/page.tsx`

도움 패널/접힘 버튼이 스크롤 컨테이너 영향을 받지 않도록 portal로 body에 렌더링해 항상 우측 하단 고정되도록 했습니다. `src/components/HelpPanel.tsx`

설정 페이지의 프로필/워크스페이스 블록은 section 전역 패딩 영향을 받지 않도록 section 태그를 div로 변경했습니다. `src/app/app/settings/page.tsx`

앱 쉘을 `h-screen overflow-hidden`으로 고정해 사이드바가 스크롤에 영향받지 않도록 했습니다. `src/components/AppShell.tsx`
