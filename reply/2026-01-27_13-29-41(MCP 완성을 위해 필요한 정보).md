# 질문
현재 서비스에서 cafe24를 통해 쇼핑몰을 운영하는 사람을 대상으로 24시간 자동 AI CS서비스를 제공하고자 합니다.
이때 사용자는 어떤 정보를 입력해야 cafe24의 주문, 고객 정보 등을 서비스에서 인지할 수 있도록 할 수 있나요?

# 답변
Cafe24의 주문/고객 정보를 읽어오려면 일반적으로 “쇼핑몰 연동(앱/외부서비스 연결)”을 위한 인증 정보가 필요합니다. 보편적인 흐름은 아래처럼 구성됩니다.

필수 입력(또는 연결 단계에서 확보해야 하는 것)
- 쇼핑몰 도메인: 예) yourmall.cafe24.com 또는 보유 도메인
- 카페24 앱 연동 권한 동의(관리자 로그인 기반 OAuth): 주문, 회원/고객, 배송, CS 관련 스코프
- 앱 인증 정보: Client ID / Client Secret (서비스가 Cafe24 개발자센터에서 발급받은 값)
- 콜백/리다이렉트 URL 등록: OAuth 완료를 위한 리다이렉트 주소
- 상점 식별자: mall_id(카페24에서 사용하는 상점 ID)

운영 중 필수로 안전하게 저장/관리해야 하는 정보(사용자 입력이 아니라 OAuth 완료로 확보)
- Access Token / Refresh Token
- 토큰 만료 시간(expires_at) (https://developers.cafe24.com/docs/api/admin/#apps 문서의 기준에 따라 자동 반영)

선택 입력(서비스 품질 향상)
- CS 기본 정보: 고객 응대 정책, 반품/교환 규정, 배송사/배송정책
- 채널 연동 범위: 카카오톡/톡상담/이메일 등

권장 UX
1) 사용자는 “쇼핑몰 주소(mall_id)”만 입력
2) “카페24 로그인 → 권한 동의” 버튼으로 OAuth 진행
3) 서비스가 토큰과 권한을 저장하고, 이후 주문/고객 정보를 주기적으로 동기화

정리
사용자는 ‘쇼핑몰 식별 정보 + 카페24 로그인(권한 동의)’만 제공하면 되고, 실제 주문/고객 데이터 접근에 필요한 토큰은 OAuth 과정에서 자동으로 확보/저장하는 구조가 가장 안전하고 일반적입니다.
