# 질문
1. 개인화된 템플릿과 도메인을 사용하고자 합니다. (supabase 템플릿은 스팸함으로 보내어집니다. 도메인을 사용하더라도 가입시 스팸함을 확인하라는 안내문구가 있어야 합니다.)
2. 인증 완료 후 /login 페이지로 리디렉션되도록 하면 됩니다.
3. 이메일 인증 완료 후 사용자에게 값을 받아 생성하면 됩니다.

# 답변
Supabase Auth 이메일 인증 가입을 위한 프론트 연동을 완료했습니다.

적용 내용
- 가입: 이메일/비밀번호 → 인증 메일 발송 (redirect: /login)
- 로그인: 이메일 인증 완료 계정만 로그인 허용
- 가입 화면에 스팸함 확인 안내 문구 추가
- Verify 화면 안내 문구로 전환
- RequireAuth, ProfileMenu, Settings에서 Supabase 세션 사용

남은 설정(콘솔)
1) Supabase Auth > URL Configuration
   - Site URL: https://mejai.help
   - Redirect URLs: https://mejai.help/login (로컬 개발 시 http://localhost:3000/login 추가)
2) Supabase Auth > Email Templates
   - 커스텀 템플릿/브랜딩 적용
3) Supabase Auth > SMTP Settings
   - 커스텀 도메인 SMTP 연결

필요한 환경변수
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
