# 답변
세션 상세 화면에 **통화 처리 단계 트리거 UI**를 추가했습니다.
이제 `/app/calls/{id}`에서 ASR/요약/확인/최종 응답/이관을 클릭하면
`/api/sessions/{id}/process`를 호출해 **turns + event_logs + (이관 시) review_queue**가 생성됩니다.

변경 파일
- `src/app/app/calls/[sessionId]/page.tsx`
- `docs/단계별_구현_체크리스트_및_검증.md` (상세 항목 기록)

사용 방법
1) `/app/calls` → 세션 상세 진입
2) “세션 작업” 영역의 버튼을 순차적으로 클릭
   - ASR 기록 → 요약 기록 → 확인 요청 → 최종 응답
   - 필요 시 “이관 처리” 클릭
3) 우측 “세션/턴 로그”와 “대화 타임라인”이 즉시 갱신되는지 확인

다음 단계로 원하시면 **실시간 흐름(순차 자동 호출)** 또는 **실제 WebSocket 이벤트 수신 처리**로 확장하겠습니다.
